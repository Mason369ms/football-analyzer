🔧 改进详情

问题 1: 赛事列表重复 ✅ 已解决

根本原因：使用 INSERT 语句导致同一场比赛可插入多条分析记录

解决方案：

  • 修改 record_analysis() 函数为更新逻辑
  • 使用 INNER JOIN 确保"赛事列表有分析才会有"
  • 清理历史重复数据（删除 6 条重复记录）

问题 2: 新增序号列 ✅ 已完成

数据来源：竞彩 API 的 lottery_info.round 字段（如 "周四201"）

实现方式：

  • 数据模型添加 round_info 字段
  • 查询时关联 matches.round_info
  • 提取数字部分作为序号（201, 202 等）
  • 赛事列表和近期分析都新增序号列
关键代码改动： # history_db.py - record_analysis() existing = conn.execute( "SELECT id FROM analyses WHERE match_id = ?", (match_id,) ).fetchone() if existing: # ✅ 更新现有记录 conn.execute("UPDATE analyses SET ... WHERE match_id = ?") else: # ✅ 插入新记录 conn.execute("INSERT INTO analyses ...") # dashboard.py - render_dashboard_html() match_number = "" if m.round_info: import re num_match = re.search(r'(\d+)', m.round_info) # ✅ 提取序号：周四201 → 201 match_number = num_match.group(1)
